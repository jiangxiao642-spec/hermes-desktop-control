# v3.7 实战测试 — 2026-05-25

## 测试发现

### Store 应用激活

- Claude Desktop (Store版) COM 启动后窗口不可见：GetWindowRect 全零，IsWindowVisible=False
- Win32 ShowWindow/SetForegroundWindow 全线返回 False
- UIA FindFirst(Edit control) 返回 null
- **结论：** 降级链逻辑对，但 Store 沙箱比预期更严。当前唯一可靠激活方式：用户手动点击窗口。

### 粘贴验证三步

- 记事本（Win32）测试：ShowWindow/SetForegroundWindow 仍返回 False
- ^v 发送后 500ms 截图→vision 验证内容未进入
- **根因：** Win32 激活 API 在 WSL 桥接环境下普遍假失败
- **结论：** 粘贴验证规则正确，但依赖窗口能获得焦点。Store 应用需用户辅助。

### 视觉读回复

- 三次截图读 Claude Desktop 回复，vision 均返回用户消息而非 assistant 回复
- **根因：** 未先滚底（{END}），且 vision 未区分消息左右侧
- v3.7 规则已覆盖此问题，但测试时未执行滚动步骤

## 影响

- Store 应用（Claude Desktop）全自动化不可行
- Win32 应用激活在 WSL 桥接下不可靠
- 视觉读取需严格执行"滚底→截图→从下往上找非己消息"规则
