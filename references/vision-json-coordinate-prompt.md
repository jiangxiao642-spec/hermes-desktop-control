# 视觉坐标直出 prompt 模板

## 症结

旧流程：`qwen看图 → 写作文 → DeepSeek读作文 → 猜坐标`，两段误差。
新流程：`qwen看图 → 报JSON坐标 → 叠加裁剪偏移 → click`，只剩视觉定位精度。

## Prompt 模板

```
图片尺寸{s}×{s}像素，找到「{target}」的中心点，
返回相对于本图左上角的像素坐标。
JSON格式：{{"x": 像素值, "y": 像素值, "confidence": 0-1}}
找不到目标时返回：{{"x": -1, "y": -1, "confidence": 0}}
不要输出其他任何内容。
```

模板变量：
- `{s}`: 裁剪尺寸，默认 200
- `{target}`: 目标元素描述（如"发送按钮""微信输入框"）

## 坐标系对齐

qwen 返回的是裁剪图相对坐标。真值 = crop_left + x, crop_top + y。

例：裁剪左上角 (340, 520)，qwen 返回 (45, 87) → 屏幕坐标 (385, 607)。

## 已排除的替代方案

### 让 qwen 直接返回绝对屏幕坐标
不可行：qwen 不知道自己在整张屏幕上的位置。
结论：必须走相对坐标 + 外层偏移的模式。

### 传给 qwen 屏幕坐标上下文
图片尺寸和裁剪左上角可以在 prompt 里说明，但不强制。

## 容错规则

qwen 返回有以下几种形式：
1. `{"x": 85, "y": 18, "confidence": 0.95}` — 裸 JSON ✅
2. ` ```json\n{"x": 85, ...}\n``` ` — markdown 包裹 ✅
3. `图片中未找到...` — 目标不存在，退回自然语言 ❌
4. 空内容 / 非 JSON — API 异常 ❌

兜底 prompt 行 `"找不到目标时返回：{\"x\": -1, ...}"` 应消除情况 3 和 4，使所有返回都可被 JSON 提取层统一处理。实测待验证。
