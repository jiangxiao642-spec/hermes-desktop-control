## Chrome_WidgetWin_1 激活问题

Edge、Claude Desktop、Settings 均使用 Chrome_WidgetWin_1 或 ApplicationFrameWindow。
ShowWindow + SetForegroundWindow 频繁超时（10s），返回 True 但窗口未到前台。

**解决：** 跳过 Win32 激活，直接用 UIA SetFocus 聚焦目标控件。
SetFocus 对 ProseMirror（Claude Desktop）和 Edit（Edge URL栏）可靠。

## ProseMirror 粘贴时序

Claude Desktop 的 Claude_pzs8sxrjxfjjc!Claude（Store 版）：
- UIA SetFocus 可聚焦
- ^v 粘贴仅在窗口真前台时生效
- 窗口后台时 ^v 不响应，ValuePattern 读回仍为 "Write a message..."
- `[System.Windows.Forms.SendKeys]::SendWait('^v')` 在部分后台场景比 bridge send_keys 可靠
- ValuePattern.SetValue 中文长文本（>70字）超时

**可靠流程：**
```
1. UIA 定位 ProseMirror editor (Name="Write your prompt to Claude")
2. SetFocus（不超时）
3. clipboard_write 文本
4. 确认窗口在前台（手动 Alt+Tab 或任务栏点击）
5. send_keys ^v（分步，不跟 {ENTER} 合并）
6. ValuePattern 读回验证
7. send_keys {ENTER}（单独调用）
```

## 回复读取

Claude Desktop 的 ProseMirror 回复被拆成大量小 Text element（10+个），
每个几十字符。按 Y 坐标过滤 assistant 消息区域后拼接重构。

示例范围：Y=840-1420 包含一条完整回复的 8+ 个 Text 碎片。

## iLink 微信通道

- 文件传输助手不在 bot 联系人列表时 `ret=-3`
- 需在微信客户端里向 bot 发消息建立会话
- 连续 3+ 次调用触发 `ret=-2 rate limited`
